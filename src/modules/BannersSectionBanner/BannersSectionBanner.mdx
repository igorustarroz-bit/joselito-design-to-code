# Banners / Section banner

## Intro

`BannersSectionBanner` traduce el módulo **Banners / Section banner** de Figma (`node 58182:4380`, COMPONENT_SET: Property1=desktop/mobile). Cabecera de sección + banner de imagen con tarjeta de texto.

Es un módulo "contenedor" (`src/modules/`, no `src/components/`): ocupa el 100% del ancho del viewport.

## Demo

<Canvas of={BannersSectionBannerStories.Default} />

## Anatomía

Cabecera (reutiliza el componente `Title` tal cual: título + enlace "Ver todos" con chevron) + banner de imagen con título/párrafo/botón.

## Comportamiento

**Diferencia estructural real entre breakpoints** (no solo un reescalado visual, verificado con `get_design_context`): en mobile la imagen (`AspectRatio` `ratio="4:3"`) va a ancho completo con el texto/botón apilados debajo; en desktop la imagen (`ratio="16:9"`) ocupa toda la altura de la sección y una tarjeta blanca se superpone en la esquina inferior derecha. Se renderizan ambas estructuras completas y se alternan por media query — mismo criterio ya usado en Content / Intro text, Cards / Gallery, Cards / Showcase y Cards / Categories.

**Botón con tamaño discreto por breakpoint** (`size="s"` mobile / `"m"` desktop), mismo criterio que Content / Intro text.

## Tokens

- Cabecera: mismos tokens ya documentados en `Title.mdx` (`--text-body-3`/`--text-body-4` responsive).
- Título de tarjeta: `--font-title`/`--text-title-2`/`--tracking-title` (28px mobile/32px desktop, coincide exacto con la escala responsive).
- Texto de tarjeta: `--text-body-3`, color `--color-content-neutral-2` (`#999`).
- Padding de sección: `--spacing-fx-6`/`--spacing-fx-9` (mobile, primitivas fijas), `--spacing-r-13`/`--spacing-r-9` (desktop, responsive reales verificados en 1440px: 80px/48px).
- Ancho/padding de tarjeta desktop: 372px literal, `--spacing-fx-8` (padding).
