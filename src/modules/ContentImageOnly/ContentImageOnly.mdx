# Content / Image only

## Intro

`ContentImageOnly` traduce el módulo **Content / Image only** de Figma (`node 58163:40001`, COMPONENT_SET: Images=1/2 × Device=desktop/mobile). Módulo de solo imagen(es), sin ningún texto — el más simple de los módulos "Content" junto a Text only y Text + Image.

Es un módulo "contenedor" (`src/modules/`, no `src/components/`): ocupa el 100% del ancho del viewport.

## Anatomía

- **1 imagen**: una única `AspectRatio` (`ratio="16:9"`) a todo el ancho del módulo.
- **2 imágenes**: dos `AspectRatio` (`ratio="3:4"` cada una) en fila, con anchos asimétricos fijos (una más estrecha que la otra, replicando las instancias de ejemplo de Figma).

## Comportamiento

La propiedad "Device" de Figma (desktop/mobile) **no se tradujo como prop**: es responsive real, resuelto con una media query en `768px` (mismo criterio que Content / Text only y Content / Text + Image). La propiedad "Images" (1/2) sí es una decisión de disposición real, traducida como la longitud del array `images` — con 2 elementos se activa el layout de dos columnas, con 1 (o 0) el de una sola imagen a todo el ancho.

Se verificó en Figma que la variante de 2 imágenes cambia de comportamiento entre breakpoints, no solo de tamaño: en desktop usa `justify-content: space-between` (las dos imágenes con anchos fijos de 424px/648px, separadas por el espacio restante del contenedor) mientras que en mobile usa un `gap` fijo de 12px con alineación a la izquierda (anchos fijos de 142px/196px) — se respetó esa diferencia de comportamiento tal cual, en vez de forzar el mismo mecanismo de espaciado en ambos breakpoints.

## Variantes y tamaños

| Prop `images` | Layout |
|---|---|
| 1 elemento (o vacío) | 1 imagen, ratio 16:9, a todo el ancho |
| 2 elementos | 2 imágenes, ratio 3:4 cada una, anchos asimétricos fijos (424px/648px desktop, 142px/196px mobile) |

## Tokens

Ningún token nuevo — reutiliza `--color-surface-base` para el fondo, ya existente. Los paddings verticales de sección (40px/60px) no están enlazados a ninguna variable en Figma y se mantienen como literales.

**Retrofit grid de 12 columnas**: en la variante de 2 imágenes, 424px y 648px (desktop) son exactamente 4 y 6 columnas (4×88+3×24=424, 6×88+5×24=648); el hueco que producía `justify-content: space-between` (~248px) es exactamente 2 columnas vacías + sus 3 gutters (2×88+3×24=248) — hueco de columna real, no arbitrario. Se reescribió como grid real de 12 columnas: imagen 0 = `grid-column: span 4`, imagen 1 = `grid-column: 7 / span 6` (salta las columnas 5-6). En mobile, 142px/196px ≈ 5 y 7 columnas (5+7=12, sin hueco), con el gap de 12px ya coincidiendo exactamente con `--grid-gutter` en ese breakpoint — también expresado como grid real (`span 5` + `span 7` contiguos). El margen lateral de esta variante pasó de literal a `padding-inline: var(--grid-margin-default)` (mismo valor numérico). La variante de 1 imagen no tiene margen lateral en ningún breakpoint (edge-to-edge verificado) y se dejó sin `padding-inline` a propósito.

## Propiedades

| Nombre | Descripción | Por defecto | Control |
|---|---|---|---|
| `images` | Array de `{ src, alt }` (1 o 2 elementos deciden el layout) | `[]` | — |

## Accesibilidad

- Usa `<section>` como raíz — landmark semántico correcto para un bloque de contenido de página.
- Cada imagen usa `alt` (vacío por defecto, indicado explícitamente por quien use el módulo si es informativa).

## Componentes relacionados

- **Aspect Ratio** — caja de imagen (`ratio="16:9"` con 1 imagen, `ratio="3:4"` con 2).
- **Content / Text + Image** / **Content / Text only** — módulos hermanos de la misma familia "Content".
