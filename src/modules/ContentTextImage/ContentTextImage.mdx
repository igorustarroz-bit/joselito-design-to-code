# Content / Text + Image

## Intro

`ContentTextImage` traduce el módulo **Content / Text + Image** de Figma (`node 58163:39941`, COMPONENT_SET con 3 variantes: Device=desktop×Layout=text-right, Device=desktop×Layout=text-left, Device=mobile×Layout=text-all). Marcado en las instrucciones del proyecto como referencia de calidad — la mejor construida en Figma, usada aquí como base de comparación para el resto de módulos.

Es un módulo "contenedor" (`src/modules/`, no `src/components/`): ocupa el 100% del ancho del viewport.

## Anatomía

1. **Imagen** (ratio 2:3) — a la izquierda o derecha según `layout`
2. **Bloque de texto**: label opcional + título (título de titular) + párrafo + botón opcional (reutiliza `Button` de UI02)

En mobile (por debajo de 768px de ancho) se apila siempre en columna, con la imagen arriba — independientemente del valor de `layout`.

## Comportamiento

La propiedad "Device" de Figma (desktop/mobile) **no se tradujo como prop**: es responsive real, resuelto con una media query en `768px` (el mismo breakpoint "m" en el que ya cambia `--grid-margin-default` en `responsive.css`) — no son dos componentes distintos. La propiedad "Layout" (text-left/text-right) sí es una decisión de diseño real y se tradujo como la prop `layout`: decide en qué lado aparece la imagen a partir de 768px, resuelto con `order` de flexbox (el orden en el DOM no cambia).

## Tokens

Ningún token nuevo — reutiliza globales ya existentes, verificado literalmente en `get_design_context` de las 3 variantes:

| Elemento | Token |
|---|---|
| Fondo | `--color-surface-base` |
| Label | `--text-body-6` / `--text-body-6--line-height` (idéntico en mobile y desktop) |
| Título | `--font-title`, `--text-title-4`, `--tracking-title` (idéntico en mobile y desktop) |
| Párrafo | `--font-body`, `--text-body-5` / `--text-body-5--line-height` |
| Gap título/párrafo | `--spacing-fx-8` (32px, desktop) / `--spacing-fx-6` (24px, mobile) |
| Gap label/título | `--spacing-fx-4` (16px, desktop) / `--spacing-fx-2` (8px, mobile) |

Los paddings verticales de sección (40px/60px en mobile; 120px en desktop) no están enlazados a ninguna variable en Figma y se mantienen como literales. El padding horizontal (20px mobile / 60px desktop) sí se centralizó como `padding-inline: var(--grid-margin-default)` en el retrofit de grid de 12 columnas — mismo valor visual, ahora sobre el token compartido en vez de un literal.

**Retrofit grid de 12 columnas**: la fila desktop imagen+contenido se expresa con anchos equivalentes por `calc()` (`--col-unit`) en vez de `flex-grow`/`width` fijo: imagen = 5 columnas (5×88+4×24 = 536px a 1440px, coincide exactamente con la medida real de Figma), contenido = 6 columnas (6×88+5×24 = 648px, también exacto). El gap real entre ambos (60px) no coincide con `--grid-gutter` (24px a este breakpoint) ni con un hueco de columna vacía limpio, así que se mantiene como literal `60px` documentado como excepción editorial deliberada, no como bug. Nota: el CSS anterior combinaba `flex: 1 0 0` en la imagen con `justify-content: space-between` y un `gap: 60px` explícito — esa combinación en realidad ignoraba el gap literal y repartía todo el espacio libre como gap (~136px a 1440px en vez de 60px), y además la imagen crecía a ~612px en vez de los 536px documentados. El retrofit a anchos fijos por columna corrige ese desajuste no detectado previamente.

**Inconsistencia detectada** (documentada, no propagada): en la variante mobile, el párrafo de texto está enlazado en Figma a un estilo huérfano ("Title/medium", familia `static/font/brand` = "Neue Haas Grotesk Display Pro", 16px/24px) completamente distinto de la familia `Tipography/*` usada en el resto del sistema — incluida la propia variante desktop de este módulo, que sí usa Body/05 correctamente. Se implementó con `--font-body` + `--text-body-5` en ambos breakpoints por consistencia, en vez de introducir una familia tipográfica nueva para un único párrafo de un único módulo.

**Nota sobre el botón**: la instancia de `UI02-Button` en Figma mezcla altura de tamaño `L` (64px) con padding de tamaño `M` (24px) — una probable anulación manual de la instancia en este módulo concreto, no una variante sistemática. Se usó el componente `Button` ya construido con `size="m"` tal cual, sin crear un tamaño ad-hoc para esta única instancia.

## Accesibilidad

- Usa `<section>` como raíz — landmark semántico correcto para un bloque de contenido de página.
- El botón reutiliza `Button`, ya accesible (foco, teclado, `:focus-visible`).
- La imagen usa `alt` (vacío por defecto, indicado explícitamente por quien use el módulo si es informativa).
